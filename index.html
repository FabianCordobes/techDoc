What is CSS?

CSS, which stands for Cascading Style Sheets, is a declarative language that controls the appearance of web pages in the browser. The browser applies CSS style declarations to selected elements to display them correctly. A style declaration contains the properties and their respective values, which determine how a web page will look.

CSS is one of the three main web technologies, along with HTML and JavaScript. CSS usually styles (HTML elements), but it can also be used with other markup languages ​​like SVG or XML.

The term "cascading" refers to the rules that determine how selectors are nested when changing the appearance of a web page. This is a very important feature, as a complex website can contain thousands of CSS rules.

Specifity

Specificity is one of the key concepts of CSS. It is the way in which browsers resolve conflicts if several rules are applied to the same element, deciding which styles are most relevant and therefore which ones will be applied.

Unlike other languages, CSS doesn't throw an error or break if it encounters a line it can't parse while reading. The CSS cascade reads from top to bottom, filters and classifies the rules according to their importance, if it cannot parse a value it will take it as invalid and therefore ignore it.

What happens when there is a match of elements? When there is a CSS conflict, it takes the last value written. If the conflict is in the importance of the styles, the browser will make a calculation to resolve which style is more relevant.

How is the score taken? The more specific a style is, the more likely it will be applied. For this, the browser determines by means of "weights" or scores which style is the most relevant for an element.

Weight category selector

The selector weight categories are listed here in decreasing order of specificity:

!important 1,0,0,0,0

  • Rewrite any style.
  • For the browser it is of high priority.
  • It must be avoided or used with great caution.

Online styles 0,1,0,0,0

  • Inline styles will apply new styles over others, even if they already have classes or IDs.

#ID 0,0,1,0,0

  • IDs have more weight than classes and labels.
  • But since they are unique, it is preferable to use them for functionality than for styles.

.Classes, attributes and pseudo classes 0,0,0,1,0

  • It's better to use classes to give styles, combined with a methodology like BEM you will have a manageable specificity.

Elements 0,0,0,0,1

  • The styles applied to elements are the most basic and easy to rewrite.

Universal Selector

  • The universal selector has no weight and will be overridden by any other rule.
Basic Box-Model

When styling a document, the browser's rendering engine renders each element as a rectangular box according to the standard CSS basic box model. CSS determines the size, position, and properties (color, background, border size, etc.) of these boxes.

Each box is made up of four parts (or areas), defined by their respective borders: the content border, the padding border, the border border, and the margin border.

Content area

The content area, bounded by the content edge, contains the "real" content of the element, such as text, an image, or a video player. Its dimensions are the content width (or content-box width) and the content height (or content-box height). It often has a background color or background image.

If the box-sizing property is set to content-box (default) and if the element is a block element, the content area's size can be explicitly defined with the width, min-width, max-width, height, min-height, and max-height properties.

Padding area

The padding area, bounded by the padding edge, extends the content area to include the element's padding. Its dimensions are the padding-box width and the padding-box height.

The thickness of the padding is determined by the padding-top, padding-right, padding-bottom, padding-left, and shorthand padding properties.

Border area

The border area, bounded by the border edge, extends the padding area to include the element's borders. Its dimensions are the border-box width and the border-box height.

The thickness of the borders are determined by the border-width and shorthand border properties. If the box-sizing property is set to border-box, the border area's size can be explicitly defined with the width, min-width, max-width, height, min-height, and max-height properties. When there is a background (background-color or background-image) set on a box, it extends to the outer edge of the border (i.e. extends underneath the border in z-ordering). This default behavior can be altered with the background-clip CSS property.

Margin area

The margin area, bounded by the margin edge, extends the border area to include an empty area used to separate the element from its neighbors. Its dimensions are the margin-box width and the margin-box height.

The size of the margin area is determined by the margin-top, margin-right, margin-bottom, margin-left, and shorthand margin properties. When margin collapsing occurs, the margin area is not clearly defined since margins are shared between boxes.

Finally, note that for non-replaced inline elements, the amount of space taken up (the contribution to the height of the line) is determined by the line-height property, even though the borders and padding are still displayed around the content.
